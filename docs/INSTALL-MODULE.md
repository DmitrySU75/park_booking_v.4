# Установка и удаление модуля avs_booking

## Системные требования

| Компонент    | Минимальная версия |
| ------------ | ------------------ |
| PHP          | 7.4                |
| Битрикс      | 20.0.0             |
| MySQL        | 5.6                |
| LibreBooking | 2.8                |

### Необходимые расширения PHP

- curl
- json
- mbstring
- pdo_mysql

## Установка модуля

### Шаг 1. Подготовка

1. Убедитесь, что установлен и настроен **LibreBooking** (см. [инструкцию по установке LibreBooking](https://librebooking.readthedocs.io/))
2. Создайте в LibreBooking:
   - Расписания (почасовое, дневное, ночное)
   - Кастомный атрибут «Тип аренды»
   - Ресурсы (беседки)

### Шаг 2. Копирование файлов

Скопируйте папку `avs_booking` в директорию:
/путь/до/сайта/local/modules/avs_booking/

### Шаг 3. Установка через админку

1. Авторизуйтесь в административной панели Битрикс
2. Перейдите: **Настройки** → **Настройки продукта** → **Модули**
3. Нажмите кнопку **«Обновить список модулей»**
4. Найдите модуль **`avs_booking`** в списке
5. Нажмите **«Установить»**

### Шаг 4. Настройка модуля

После установки перейдите в:

**Настройки** → **Настройки продуктов** → **Настройки модулей** → **AVS Booking**

Заполните следующие вкладки:

#### Вкладка «API LibreBooking»

| Поле                       | Значение                                              |
| -------------------------- | ----------------------------------------------------- |
| URL API                    | `https://ваш-домен.ru/booking/Web/Services/index.php` |
| Логин API                  | `libreuser`                                           |
| Пароль API                 | ваш пароль от LibreBooking                            |
| ID расписания по умолчанию | `2`                                                   |
| Часовой пояс               | `+05:00` (для Екатеринбурга)                          |

#### Вкладка «Оплата»

| Поле                          | Значение                         |
| ----------------------------- | -------------------------------- |
| Сумма предоплаты по умолчанию | `0`                              |
| Shop ID ЮKassa                | из личного кабинета ЮKassa       |
| Секретный ключ ЮKassa         | из личного кабинета ЮKassa       |
| ID платежной системы          | `2` (после создания в Битрикс)   |
| ID товара «Аренда беседки»    | создайте товар-услугу в каталоге |

#### Вкладка «Уведомления»

| Поле                 | Значение                 |
| -------------------- | ------------------------ |
| Email администратора | admin@example.com        |
| Webhook Битрикс24    | URL вебхука из Битрикс24 |

#### Вкладка «API для 1С»

| Поле            | Значение                                           |
| --------------- | -------------------------------------------------- |
| API-ключ для 1С | скопируйте сгенерированный ключ (используйте в 1С) |

#### Вкладка «Сезонные настройки»

| Поле                          | Значение |
| ----------------------------- | -------- |
| Начало летнего сезона         | `01.06`  |
| Конец летнего сезона          | `31.08`  |
| Зимнее время окончания работы | `22`     |
| Летнее время окончания работы | `23`     |

### Шаг 5. Добавление компонента в шаблон

В шаблон детальной страницы беседки (обычно `detail.php`) добавьте:

```php
<?php
if (\Bitrix\Main\Loader::includeModule('avs_booking')) {
    $APPLICATION->IncludeComponent(
        'avs_booking:booking.form',
        '.default',
        ['ELEMENT_ID' => $arResult['ID']],
        null,
        ['HIDE_ICONS' => 'Y']
    );
}
?>
```

### Шаг 6. Настройка платежной системы ЮKassa

Перейдите: Магазин → Настройки → Платежные системы

Нажмите «Добавить платежную систему»

Выберите обработчик «ЮKassa»

Укажите:

Название: ЮKassa

Активность: Да

Тип платежной системы: Виджет ЮKassa

Shop ID: из личного кабинета

Секретный ключ: из личного кабинета

Сохраните и запишите ID платежной системы (укажите его в настройках модуля)

### Шаг 7. Настройка вебхука ЮKassa

В личном кабинете ЮKassa укажите URL для уведомлений:

```text
https://ваш-домен.ru/bitrix/tools/sale_ps_result.php
```

### Шаг 8. Очистка кэша

```bash
php /путь/до/сайта/bitrix/modules/main/cli/clear_cache.php
```

### Шаг 9. Проверка

Откройте страницу любой беседки

Должна отображаться форма бронирования

Выберите дату — должны появиться доступные типы аренды

Заполните данные и нажмите «Перейти к оплате»

## Удаление модуля

### Шаг 1. Перейдите в список модулей

Настройки → Настройки продукта → Модули

### Шаг 2. Найдите модуль avs_booking

Нажмите «Удалить»

### Шаг 3. Выберите опцию удаления

При удалении появится выбор:

| Опция            | Результат                                                                     |
| ---------------- | ----------------------------------------------------------------------------- |
| Сохранить данные | Инфоблоки «Услуги и скидки», «Ценовые периоды» и настройки модуля сохраняются |
| Удалить всё      | Полное удаление всех данных модуля                                            |

### Шаг 4. Подтверждение

Нажмите «Удалить»

### Шаг 5. Очистка кэша

```bash
php /путь/до/сайта/bitrix/modules/main/cli/clear_cache.php
```

# Устранение неполадок
| Проблема |	Решение |
| ---------------- | ----------------------------------------------------------------------------- |
| Модуль не виден в списке |	Проверьте путь /local/modules/avs_booking/install/index.php и права на файлы |
| Ошибка 500 при установке |	Включите display_errors в install/index.php, проверьте логи PHP|
| Форма не отображается |	Проверьте вызов компонента в шаблоне, очистите кэш |
| Типы аренды не показываются |	Заполните цены в свойствах беседки (PRICE_HOUR, PRICE, PRICE_NIGHT)|
| Ошибка оплаты 500 |	Проверьте настройки ЮKassa, включите логирование |
|Заказ не найден при оплате |	Проверьте ID платежной системы в настройках модуля |

## При смене домена
Обновите URL API в настройках модуля
Обновите LIBREBOOKING_BASE_URL в /local/php_interface/librebooking_config.php
Обновите URL вебхука в личном кабинете ЮKassa
Очистите кэш Битрикс

```bash
php /путь/до/сайта/bitrix/modules/main/cli/clear_cache.php
rm /путь/до/сайта/upload/booking_token.json
```

## Обновление модуля
